## Recap

### 접근법

처음에 sort (O(NlogN))을 통해 index의 범위를 제한하고 brute force를 구현했는데, 시간초과에 걸렸다.

Binary Search로 해결.

### 시간복잡도

O(N log N)

### 공간복잡도

### Missed Points

- Binary Search

Binary Search를 구현하면서, 문제 조건을 만족시키는 로직에서 실수를 많이했다.

탈출조건 & 문제조건 잘 보고 풀어야함.

### 얻어갈 점

- C++ sort (내림차순)

```sort(v.begin(), v.end(), greater<int>());```

- Binary Search

start와 end에서 +1 or -1 까먹지 않기.

## Solution Code

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

void solve();

int N, M;
vector<int> v;
int ans = -1;

int main() {
    freopen("input.txt", "r", stdin);

    cin >> N >> M;
    for (int i = 0; i < N; i++) {
        int n;
        cin >> n;
        v.push_back(n);
    }

    solve();

    cout << ans << "\n";
}

void solve() {
    // sort
    sort(v.begin(), v.end(), greater<int>());

    // binary search
    int start, end, mid;
    start = 0;
    end = v[0];
    while (start <= end) {
        mid = (start + end) / 2;

        long long sum = 0;
        for (int i = 0; i < N; i++) {
            if (v[i] > mid) {
                sum += v[i] - mid;
            }
        }

        if (sum == M) {
            ans = mid;
            break;
        } else if (sum > M) {
            if (mid > ans) {
                ans = mid;
            }
            start = mid + 1;
        } else {
            end = mid - 1;
        }
    }
}
```
