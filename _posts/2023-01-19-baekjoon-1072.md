## Recap

### 접근법

단순히 binary search를 이용해 풀 수 있는 문제지만, 실수하기 너무 좋은 문제였다.

### 시간복잡도

O(nlogn)

### 공간복잡도

### Missed Points

#### 1. Type Casting

C++ 은 나누기에서 divider의 type을 따라간다. 또한, 나누기 과정에서의 type에 의한 오류를 줄이기 위해 explicit하게 type casting을 하는 것이 좋을 듯.

```cpp
long long Z = ((long long)Y * 100) / (long long)X;
```

#### 2. Boundary Contition

>왜 틀렸는지 이유를 모를 때 가장 좋은 방법은 boundary condition을 대입해 보는 것이다.

이 문제에서는 $1<= X <= 1000000000$ and $0<= Y <= X$로 주어져 있으므로, {0, 1}, {1000000000, 0}, {1000000000, 999999999}를 대입하여 확인하는 것이 타당하였으나 이를 분석하지 않아 꽤 오랜시간을 헤멨다.


### 얻어갈 점

## Solution Code

```cpp
#include <iostream>

using namespace std;

void solve();

int X, Y;
int ans = 1000000001;

int main() {
    freopen("input.txt", "r", stdin);

    cin >> X >> Y;

    solve();

    cout << ans << "\n";
}

void solve() {
    long long Z = ((long long)Y * 100) / (long long)X;
    if (Z >= 99) {
        ans = -1;
        return;
    }

    // 최대 100000000까지의 횟수가 나올 수 있음
    int start = 1;
    int end = 1000000000;
    int mid;

    while (start <= end) {
        mid = (end + start) / 2;

        long long newZ = ((long long)(Y + mid) * 100) / (long long)(X + mid);
        // cout << mid << " " << newZ << "\n";

        if (newZ >= Z + 1) {
            if (ans > mid) {
                ans = mid;
            }
            end = mid - 1;
        } else {
            start = mid + 1;
        }
    }
    if (ans == 1000000001) {
        ans = -1;
    }
}
```
