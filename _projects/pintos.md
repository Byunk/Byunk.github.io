## Instructional OS for KAIST CS330

pintOS is composed of 4 assignments; Threads, User Programs, Virtual Memory, and File System. It requires difficult and complex problems and highly demanding implementation. Two weeks are given for each assignment, and each assignment is given about 4-50 test cases, which are cumulative.

## What I learned

### How to cooperate with others

Cooperating with others is the most difficult part of the project. It is important to communicate with teammates and persuading and being persuaded make team more creative.

The first time, we overlooked this issue. Talk a little and write each own code. It invoked a lot of problems and we had to spend much more time to revert this problem.

Also, pintOS requires us to think the logic through. So, it was very useful to talk a lot and write comments and commit messages with lots of effort.

### How to deal with complicated problems

During pintOS, we have dealt with a lot of complex problems. For example, implementing a Virtual Memory System supporting Copy-on-Write is the most difficult part of the project. There were only a few skeleton codes, and all implementations are up to us.

There is a very big difference between simply thinking about how to implement and actually implementing and making all test cases work. We have to design very carefully with tons of trials and errors.

**It was a thrilling experience to decide what data structure and algorithm to use and see the code actually work as we expected.**

### Understand how the OS works

For the whole project, we have studied and implemented numerous problems that software engineers have experienced so far. Understanding how the OS works is a great help in understanding how many different programs work and how to improve these programs to work effectively.
